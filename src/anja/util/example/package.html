The package provides the classes and interfaces to create examples for the applets or applications. It includes basic interfaces for each example, a manager to handle all examples and listener and events which simplify the interaction with the user of the applet.
As many applets handle their own examples in the most different ways, this is a way to standardize this procedure. The polygon editors offer a functionality to write Polygon2Scenes to an OutputStream. This option is rather restricted and only offers to save those special scenes.
This package does not only depend on polygonal scenes but can be used to save very application specific examples (e.g. the polygons and some additional points as in PolyRobot and a hint on the specific method that should be used with this certain example and so on).
It depends on the application which information should be saved and which is unneccessary to save. Receiving an Example and restoring it is possible via the listener inferaces in this package.

Interfaces

Example

This is the main interface of the package. Every example must implement this interface. It has two main functionalities. On the one hand there is a name given for each example. It is e.g. used by the GUI to represent each example and should therefore be representative for the example itself. On the other hand the serialization offers the possibility to save whole examples via OutputStream.

ExampleListener

The interface should be implemented by all classes that want to be informed if an example changes - if the user has chosen another example - or should be saved. The class then has to register to an eg ExamplePanel before.

Classes

Adapters

Some adapters which allow to port serializable objects to byte arrays, return the Java code to create whole objects and the possibility to port byte arrays to instances of objects again.

BasicExample

This class is a simple implementation of the Example interface.

ExampleChangeEvent

The event is send to all ExampleListener objects that have been registered. It adds the possibility to additionally add an Example object to a basic event. The example can be received via getExample() method.

ExampleManager

A manager based on an ArrayList that overrides some methods to work with Example objects.

ExamplePanel

The main graphical class of the package. It adds a load and save option for Example implementing objects. Additionally ExampleListeners can register and are informed about changes or save requests. To save an example to file use the method saveExample(). The class handles its own ExampleManager. To receive a reference use the getExampleManager() method. This way new examples can be added via the application.
The debug mode prints possible exceptions to the console if enabled. The java code mode displays a button if enabled, which allows to display the java code of the current example shown in the combo box. By default the debug mode is enabled and the jc mode is disabled.

ExampleSaveEvent

The event is send to all ExampleListener objects that have been registered. It adds the possibility to additionally add a File object to a basic event. The file can be received via getFile() method.

An example:

The main class of the applet should inherit the listener interface and create an instance of the panel


public class MyApplet
        extends JApplet
        implements ExampleListener
{
    ...
    private ExamplePanel<MyAppletExample> _examplePanel = new ExamplePanel<MyAppletExample>();
    ...
    
    this.add(_examplePanel);
    _examplePanel.addExampleListener(this);
    ...
    
    ExampleManager<MyAppletExample> em = this._examplePanel.getExampleManager();
    //add some own examples  
    ...
    
    public void exampleChanged(
            ExampleChangeEvent<MyAppletExample> e)
    {
        MyAppletExample ex = e.getExample();
        //do something with the example, e.g. load into Polygoneditor
        //if the whole polygonscene is saved, use the clone() method,
        //otherwise the example in memory is changed if the scene changes      
    }

    public void exampleSave(
            ExampleSaveEvent e)
    {
        MyAppletExample ex = new MyAppletExample();
        //Fill the current scene into the example object
        // ex.p1 = ...;
        // ex.p2 = ...;
        // ex.message = ...;
        // ex.seglist = ...;
        ...
        //The save method can only be called when the example object is ready
        //The load procedure is handled by the panel itself
        _examplePanel.saveExample(e.getFile(), ex);      
    }

    ...
}

The example object could look like this:


public class MyAppletExample
        extends BasicExample
        //implements Example
{
    public Polygon2 p1 = null;
    public Polygon2 p2 = null;
    public String message = null;
    public List<Segment2> seglist = null;
    
    public MyAppletExample()
    {
        //name is inherited from BasicExample
        this.name = (new java.util.Date()).toString();
    }

    public MyAppletExample(String name)
    {
        this.name = name;
    }
}
